    • DLX Processor

    Simplified DLX Processor Architecture Diagram

    The DLX project was developed by me and my team as part of the Microelectronic Systems course at Politecnico di Torino. The goal was to design and implement a fully pipelined 5-stage RISC processor, following the DLX architecture model.

    The project covered all stages â€” from the high-level specification down to synthesis and physical design. The implementation was done in VHDL, using mainly Vivado as simulation framework, and Synopsys tools to synthesize.


    We performed a bottom-up design approach, creating smaller components to be used to build bigger components. We started from creating the ALU blocks: the adder/subtractor, logical unit, shifter, comparator. These components were all merged into a single ALU unit. We then proceeded to create the other blocks forming the datapath: the register file, the sign-extender, the instruction memory, the data memory, the branch evaluation logic. Meanwhile, we designed the control unit, responsible for generating the necessary control signals for the datapath components. Finally, we integrated all components to build a basic functioning processor.

    At this point we started to handle more complex aspects such as data hazards and control hazards, implementing techniques like forwarding, pipeline stalling and flushing to ensure correct execution of instructions. Finally, we implemented branch prediction logic with a BHT (Branch History Table) to reduce the average penalty caused by branch instructions and lower the overall CPI, and we added a data cache with write-back policy to improve memory access times.


    Each new feature caused an increase in complexity, requiring careful planning and testing to ensure correct functionality. We validated each component individualy with a testbench before integrating it into the main processor design. And then we performed integration tests to verify the correct operation of the entire system. With the help of an assembler provided by the professors, and some custom scripts, we were able to run various assembly programs (for example a program implementing the Quicksort algorithm) on the processor, verifying its correct functionality.
