    Gameplay of Pac-Man on the LandTiger Board

    The Pac-Man project was developed by me as part of the Computer Architectures course of my first year of Master at Politecnico di Torino. The goal was to design and implement a fully functional version of the classic Pac-Man game on a LandTiger Board, using the C language in the Keil uVision IDE.

    The game was completely designed from scratch, including the graphics, sound effects, and game logic. The maze that you see I designed pixel by pixel to try to match the original one as much as possible. The hardest part was actually to make everything work and to make it nice to play.

    To implement the game I had to exploit several peripherals of the LPC1768 SoC and components of the board, including:

    • The LCD display to render the game graphics, including the maze, Pac-Man, ghosts, pellets, and score.
    • The joystick to control Pac-Man's movement.
    • The timers and interrupts to manage game timing, including Pac-Man movement, ghost movement, pellet consumption, and game over conditions.
    • The DAC and the buzzer to play sound effects

    To get more specific, the whole design and implementation process involved several challenges:

    • designing the game board to seamlessly fit all its elements:
      • creating the layout of the maze so that it fit exactly 240 standard pellets and so that it could be mapped to a matrix
      • map every other object to the matrix so that it could be referenced and manipulated during gameplay

    • managing the movement of Pac-Man:
      • making it move only within the walls and not go through them
      • ensuring the movement was fluid and responsive, without sudden steps or glitches
      • managing every possible event without causing significant slowdowns or crashes

    • implementing and managing the ghost:
      • creating shortest path algorithm that was not too memory and time consuming (definitely not A*) for the ghost's movement, making it chase Pac-Man
      • increasing its speed over time or at certain points in the game
      • handling the behavior when power pellets are eaten, making the ghost vulnerable and allowing Pac-Man to eat it


    To implement such game dynamics I had to meticulously exploit several timers and in particular timer interrupts, which allowed precise control over the moment in time or event (like movement of the joystick) that triggers certain actions in the game. Moreover I had to be careful not to write too expensive code in between event, to avoid messing up the timing of the game.
